# Data Science Project Template — DataScientist Agent

A reusable project template for any data science or machine learning task, powered by the **DataScientist** GitHub Copilot agent. Drop in your data, describe your goals, and let the agent generate a full EDA → Feature Engineering → Model Building pipeline as Jupyter notebooks.

## How It Works

The `DataScientist` agent (defined in `.github/agents/data-science.agent.md`) reads your data and custom requirements, then generates structured notebooks tailored to your specific problem. Notebooks are created **one stage at a time** so you can review outputs and guide each subsequent step with real findings.

## Project Structure

```
root/
├── app/                          ← Jupyter notebooks (generated by agent)
├── artifacts/                    ← Saved pipeline and trained models (generated by agent)
│   ├── feature_pipeline.py
│   └── models/
│       └── <model_type>_<YYYYMMDD_HHMMSS>.pkl
├── data/                         ← Your data files (read-only)
├── .github/agents/
│   └── data-science.agent.md     ← DataScientist agent definition
├── ds_meta.md                    ← Your custom requirements per ML step
├── pyproject.toml
└── requirements.txt
```

## Getting Started

### 1. Add your data

Place your dataset files in the `data/` folder. The agent expects at minimum a `train.csv` file.

### 2. Set your requirements

Edit `ds_meta.md` to describe any custom requirements for each pipeline stage. The agent reads this file before generating any notebook.

```markdown
# EDA
<describe any specific analyses, columns of interest, or questions to answer>

# Feature Engineering
<describe preferred encoding strategies, features to engineer, or columns to drop>

# Modeling
<describe preferred algorithms, evaluation metrics, or any constraints>
```

Leave a section empty if you have no special requirements — the agent will apply sensible defaults.

### 3. Set up the environment

- **Python**: 3.13
- **Package manager**: [uv](https://docs.astral.sh/uv/)

```bash
# Install dependencies
uv sync

# Activate the virtual environment (Windows)
.venv\Scripts\Activate.ps1
```

## Running the Pipeline

Generate and run notebooks one stage at a time. Ideally you should build each stage on the real outputs of the previous one.

### Step 1 — EDA

```
@DataScientist generate the EDA notebook
```

Run all cells in `app/01_eda.ipynb`. Review the outputs — distributions, missing value counts, correlations, and the Key Observations summary cell — then carry those findings into the next prompt.

### Step 2 — Feature Engineering

```
@DataScientist generate the feature engineering notebook. Based on the cell outputs of the eda findings
```

Run all cells in `app/02_feature_engineering.ipynb`. This produces `data/processed_train.csv` and `artifacts/feature_pipeline.py`.

### Step 3 — Model Building

```
@DataScientist generate the model building notebook. Use the cell outputs in the feature engineering notebook to guide modeling decisions.
```

Run all cells in `app/03_model_building.ipynb`. Trained models are saved with timestamps to `artifacts/models/`.

## Artifacts

| Artifact | Description |
|----------|-------------|
| `artifacts/feature_pipeline.py` | Reusable `apply_feature_pipeline(df)` function for transforming unseen data |
| `artifacts/models/*.pkl` | Trained model files, timestamped to prevent overwrites |

## Adding Libraries

The agent will only use libraries listed in `requirements.txt`. If a required library is missing, the agent will ask for permission before installing it. To add a package manually:

```bash
uv add <package-name>
```
